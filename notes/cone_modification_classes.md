# Three Classes of Cone Modification

Source: conversation 2026-03-27. Extension of cone_modification_technology.md.

## The Distinction: Passive vs. Active Modification

Current framework: particles modify each other's cones only through interaction —
TickScheduler phase mixing when cones overlap. This is PASSIVE — a side effect of
proximity.

Active cone modification: deliberately engineering the phase structure inside a
cone, not waiting for overlap. Shaping the interference pattern intentionally.

---

## Class 1: Phase Engineering

Imposing a specific phase pattern on a cone to reshape what the particle will do.

Already done crudely:
- Lens: position-dependent phase delay on photon cone → paths converge at focal point
- Magnetic field: rotates electron spinor phase → spin precession
- Optical cavity: selected interference modes → specific photon cone structures

What the lattice adds: phase engineering on a specific geometric structure
(RGB/CMY bipartite lattice). Working WITH the grain of the underlying geometry
rather than against it. Precision limited by not knowing the geometry; knowing
the geometry removes that limit.

## Class 2: Sublattice Selection

Selectively manipulating ψ_R and ψ_L independently. These are physically distinct
in the lattice — amplitude on different sets of spatial nodes.

Possibilities:
- Suppress ψ_L, keep ψ_R → purely right-handed state
- Fix ψ_R/ψ_L ratio → fix effective mass (mass is the oscillation rate between them)
- Pure ψ_L state → neutrino-like (no right-handed component)

Cost: work required to maintain sublattice separation against Zitterbewegung,
which naturally oscillates between ψ_R and ψ_L every tick.

Implication: the boundary between particle species is less sharp than standard
physics assumes. Electron vs. neutrino is a difference in information organization
(ψ_R/ψ_L balance), not an absolute categorical distinction.

## Class 3: Cone Topology Modification

Changing the winding number of the phase field = changing electric charge.

Constraint: charge conservation = total winding number conserved. Cannot create
or destroy winding, only transfer it between cones. A machine that modifies one
particle's topology must simultaneously modify another's in the opposite direction.

This is pair creation/annihilation — which particle accelerators already do.
The lattice adds: geometric picture of winding number transfer between cones,
which may suggest more efficient engineering of these transfers.

Devices that maintain phase coherence for much longer than current decoherence times.

in TickScheduler). Design geometries that minimize unwanted cone overlaps.
